GUIDE DE DÉPLOIEMENT
// ÉTAPE 01
Fork le repo GitHub
Va sur GitHub, crée un nouveau repository ghost-academy, et uploade tous les fichiers.
git init git add . git commit -m "feat: ghost academy initial" git remote add origin https://github.com/TON_USER/ghost-academy.git git push -u origin main
// ÉTAPE 02
Activer GitHub Pages
Dans ton repo GitHub : Settings → Pages → Source → main branch → / (root). Ton site sera live en 2 minutes sur :
https://TON_USERNAME.github.io/ghost-academy/
// ÉTAPE 03
Configurer Stripe
Crée un compte stripe.com → copie ta clé publique → configure dans js/config.js
// js/config.js stripe: { publicKey: 'pk_live_TA_CLE_ICI', plans: [{ priceId: 'price_ABC123', ... }] }
// ÉTAPE 04
Ajouter un module
Dans cet admin : Modules → + Nouveau. Remplis le formulaire. Puis Export → Copier JSON → remplace data/modules.json → push sur GitHub.
git add data/modules.json git commit -m "feat: add module 08 - forensics" git push
// ÉTAPE 05
Paiement réel (backend requis)
GitHub Pages = statique. Pour encaisser vraiment, il te faut un backend. Options gratuites recommandées :
Option 1: Netlify Functions (gratuit) → netlify.com → déploie le même repo → crée functions/checkout.js Option 2: Vercel Serverless (gratuit) → vercel.com → import GitHub repo → crée api/checkout.ts Option 3: Supabase Edge Functions → supabase.com → Edge Functions → authentification + BDD incluses
// ÉTAPE 06
Domaine personnalisé
Dans GitHub Pages Settings → Custom domain → tape ton domaine. Configure chez ton registrar :
CNAME ghostacademy.io → TON_USERNAME.github.io # Ou via Netlify/Vercel (encore plus simple)
